# ADR-004: Автоматический бэкап PostgreSQL

Статус: Принято | Дата: 2026-05-23 | Автор: Кот

## Контекст

На момент аудита от 20.05 директория /root/LabDoctorM/backups/ не существовала. Бэкапы PostgreSQL отсутствовали полностью. При потере данных — полная переустановка всех БД (snablab, vpn_bot, lab_monitoring).

## Решение

Скрипт scripts/pg_backup.sh: автоматический dump всех пользовательских БД через pg_dump -Fc + gzip. Хранение в /var/backups/postgresql. Retention 7 дней (автоочистка старых). systemd timer каждые 6 часов (00/06/12/18). Мониторинг через backup_monitor.py: проверка наличия, свежести (stale >48h, critical >7 дней), размера. Сервис pg-backup добавлен в мониторинг Польши как type:systemd.

## Последствия

Положительные: полная автоматизация бэкапов, мониторинг бэкапов, retention. Отрицательные: bэкапы локальные (при падении сервера — потеря). Митигация: можно добавить rsync на удалённый сервер.
